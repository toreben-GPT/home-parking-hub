# v0.3未決事項の初期ルール

仕様書v0.3で詳細が確定していない箇所は、データを失いにくく、あとから変更しやすい次のルールで実装しています。

## 要確認の料金

- 金額未確定の駐車場も検索結果から消さない
- 金額が確定した候補の後ろへ「要確認」として表示する
- 推測金額は保存しない

## 空き実績の比較

- 選択した利用パターンに対応する「平日/土日祝」「夜/日中」のログを使用
- `空いていた`と`残りわずか`を駐車可能として数える
- 駐車可能率が高い順
- 率が同じ場合は、確認回数が多い方を優先
- 記録がない候補は、記録がある候補の後ろ
- 表示は確認回数を隠さず、少数データを過度に断定しない

## おすすめラベル

- ラベルは料金順を変更しない
- 100円以内、徒歩5分差を既定の比較範囲にする
- `停めやすさ良好`も100円以内・徒歩5分以内で、停めやすさが上位の候補だけに付ける

## 名称順

- JavaScriptの日本語照合（`Intl.Collator("ja")`）を使用
- 名称の読み仮名入力はv1仕様にないため追加しない
- 実データで期待順にならない場合は、仕様変更として読み仮名項目を検討する

## JSON復元

- 復元前に件数を表示して確認する
- 既存文字データへマージせず、全置換する
- 形式とschemaVersionが正しくないファイルは拒否する
- Cloudflare D1の無料枠で安全に処理できる件数を事前計算し、上限を超える場合は既存データを消す前に拒否する
- 写真ファイル本体はJSONへ含めない
- 写真メタデータは復元するため、対応するR2ファイルが残っている場合だけ写真を再表示できる

## 写真

- R2 Bucketは公開せず、認証済みWorker経由で表示
- 1枚10MBまで
- JPEG、PNG、WebP、HEIC/HEIFを受け付ける
- v1では画像変換・AI解析を行わない

## セッション

- 有効期限は90日
- HttpOnly / Secure / SameSite=Strict cookieを使用
- `SESSION_SECRET`を変更すると全端末の既存セッションを無効化できる

## 利用状態

- 通常の比較検索には `利用中`だけを表示
- `候補から除外`と`閉鎖`はデータ管理画面に残し、再編集できる
